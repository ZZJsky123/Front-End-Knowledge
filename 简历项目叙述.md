###  数学建模比赛

```
利用题目附件中所给的某年某段时间机场传感器记录下的地面气象观测数据， 建立能见度与地面气场观测数据的关系表达式。气象观测数据包括： 气压、 温度、 风速、 湿度、 风向，跑道能见度等。 我所做的工作建立指标与能见度之间的影响关系。

1. 我当时第一步工作是筛选指标， 这些指标不能都用这样方程太复杂。因此首先单变量分析，通过图像去观测各个指标变化与能见度之间存在的关系。 同时结合影响雾生成的因素常识：筛选出：  气压、 风速、 温度、 湿度这几个指标。

2. 第二步就是思考如何建立方程，最初的想法比较简单, 假设气象因素与能见度满足某种线性关系， 利用多元回归拟合构建这样的线性方程。这样做的问题是数据中时常会有两段时间指标数值基本上一样但是能见度确有很大差别， 相同指标上一个时间段能见度可能有1000米， 但是到了下一个时间能见度只有300米。 所有造成直接拟合效果比较差。 当时通过仔细分析并且对比各种数据，能够确认这种矛盾现象主要是因为： 气象因素只能说明此时刻符合雾产生的条件，但是能见度是雾随时间积累的结果，因此不能直接构建线性方程。根据数据的分析，得出雾一旦达成了形成条件，雾在这段时间内变化趋势一般积累能见度越来越低，然后再消散。 因此有了雾的形成在一个时间段内具有单向性这个假设。这个假设是之后 采用概率累加的方式建立能见度方程的基础。

3. 基于 Logisti方程，气象指标数据作为参数， 得到每一时刻雾概率方程，利用该方程计算出时刻点上雾形成的概率大小。我将概率称作雾因子，同时提出雾趋势因子的概念，雾趋势因子反应的是当前时刻雾的积累，包含了时间信息，它在数值上等于当前时刻的雾因子和之前所有时刻的雾因子累加，最后利用雾趋势因子与能见度数据建立多元回归， 仿真结果比价好，并且用这种方式建立的方程对雾的形成有比较好的预测作用。 
```

### Python 微信账单程序

```
策略者模式：将变化的部分A从主程序中解耦出来， 作为单独的算法库进行维护。主程序只维护一个A的引用，根据需求去选择库中的算法。在程序中我根据策略者模式的思想，将账单的数据处理和账单信息独立出去， 根据具体的账单类型选择处理方法和返回特定的账单信息。
   
观察者模式： 发布者 - 订阅者，信息发布者内部维护一个数组用来收集订阅者，当有消息之后发布者会遍历数组调用订阅者本身的更新方法进行信息更新。在程序中， 定义用户类 Person 其内部维护一个数组，用于注册各种类型账单，当需要获取最新的账单信息时，遍历数组通知账单更新信息。
```
