## cookie

```
cookie 是存儲在瀏覽器中過的純文本文件，瀏覽器安裝目祿下會有專門的
cookie文件夾存放各個域下設置的cookie

當網頁發送http請求時，瀏覽器會先檢查是否有相應的cookie，將其自動
添加到request Header 的cookie字段中，一般cookike存放的數據是
每次請求服務器端都需要的數據(身份信息)，cookie大小不超過4kB

cookie打印格式： 一串字符串，有key=value組成，鍵值之間由冒號隔開
document.cookie可以獲取非httponly的cookie

访问的网站可以将用户ID信息，密码，登录时长存储在cookie中，响应报文的
响应头中添加Set-Cookie字段，浏览器会将该Cookie存储下来，下一次访问将该Cookie返回给浏览器。

命名规则： 用户名@网站地址[数字].txt
缺点： 容量限制 <4KB
      cookie信息在请求头上带着，有数据安全风险
      cookie欺骗
```

## 本地缓存的其他方式

```
Cookie 本地缓存的一种方式

其他： Localstorage， sessionStroage， indexDB

localStroage： 本地长期存储，大小<5M ,无时间存储限制。
               数据存储在localStroage后，浏览器无需再向服务器进行请求
sessionStroage: 会话存储，存储时间不是永久的，浏览器一旦关闭则储存信息将会
                消失
indexedDB：浏览器在涉及存储大量数据场景下的解决方案，比如：在线编辑。indexedDB是浏览器本地数据库，可以提供查找接口和索引查询。

IndexedDB采用对象仓库存储数据，每一条数据记录包含一个独一无二的主键磨和数据，数据记录形式采用键值对的方式。
异步： 操作数据时不会锁死浏览器，用户依然可以进行操作
支持事务： transaction，即一旦发生错误，会将整个事务取消，回滚到上一个事务状态
打开indexedDB
var request = window.indexedDB.open(databaseName, version);
```
