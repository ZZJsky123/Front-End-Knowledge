## EventLoop

```
1. JS是单线程的，这一点是它的核心将来也不会改变。JS的单线程是由它的用途决定的，两个重要的用途
   操作DOM和与用户交互。 若JS成为多线程的，则在操作DOM多线程会引起诸多复杂问题。
   
2. 单线程意味着任务的执行需要排队，由于有的任务不能被立即执行，因此设计者为了更好的利用计算资源，将线程中的任务分为两种：同步和异步，同步任务都存在在主线程上，按顺序执行；异步指的是不进入到主线程而是待在任务队列中的任务，直到异步任务可执行，该任务会进入到主线程中等待被执行。

3. 任务队列中存放的是事件：用户事件，回调。主线程从任务队列中不停的读取任务，被称为eventloop
   任务分为两种：                (任务队列有两种 微任务 与 宏任务)
               微任务： promise
               宏任务   setTimeout，setInterval  xhr
   微任务先与宏任务执行，宏任务处理完后会触发下一的时间循环
3. 宏任务在执行时需要切换进程
   微任务执行时是进程之间的切换。
   
4. 例子：计时器是实时的不能被阻塞，所以它被放在了新的进程当中是宏任务
```
